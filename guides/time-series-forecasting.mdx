# Time Series Forecasting with SARIMAX

This guide demonstrates how to build SARIMAX (Seasonal AutoRegressive Integrated Moving Average with eXogenous variables) models for forecasting daily COVID-19 deaths. The approach uses cluster information and exogenous variables to improve predictions.

## Why SARIMAX?

SARIMAX models are suitable for COVID-19 death prediction because they:

- **Capture temporal dependencies**: Model both short and long-term patterns in time series
- **Handle seasonality**: Account for weekly and seasonal patterns in reporting
- **Incorporate exogenous variables**: Include factors like cases, vaccination rates, and government measures
- **Provide interpretability**: Understand the contribution of each factor to predictions

<Warning>
Forecasting COVID-19 deaths is challenging due to:
- **Limited data**: Only ~1,460 daily observations per country (4 years)
- **High volatility**: Reporting delays, new variants, and policy changes introduce noise
- **Non-stationarity**: The pandemic evolved through distinct phases (waves)
</Warning>

- **Per million**: Enables comparison across countries with different populations

### Create Lagged Features

Add lagged values to capture temporal dependencies:

```python
def crear_lags(df, lags_cases, lags_deaths):
    """
    Create lagged features for cases and deaths
    """
    df = df.copy()
    
    # Lags for cases
    for lag in lags_cases:
        df[f'new_cases_lag{lag}'] = df['new_cases_smoothed_per_million'].shift(lag)
    
    # Lags for deaths
    for lag in lags_deaths:
        df[f'new_deaths_lag{lag}'] = df[target].shift(lag)
    
    # Remove NaN rows created by lagging
    df = df.dropna().reset_index(drop=True)
    
    return df

# Define lags
lags_cases = [7, 14, 21]  # 1, 2, and 3 weeks
lags_deaths = [7, 14]     # 1 and 2 weeks

df_spain = crear_lags(df_spain, lags_cases, lags_deaths)
```

### Train-Test Split

Use time-based splitting (no shuffling for time series):

```python

min_train_size = 425
```

### Automatic Model Selection with auto_arima

Use `auto_arima` to find optimal SARIMAX parameters:

```python
# Find best SARIMAX parameters
print("Finding optimal SARIMAX parameters...")

auto_model = pm.auto_arima(
    y_train,
    exogenous=X_train,
    seasonal=True,
    m=7,                    # Weekly seasonality
    stepwise=True,
    suppress_warnings=True,
    error_action='ignore',
    trace=True,
    max_p=3, max_q=3,       # Non-seasonal orders
    max_P=2, max_Q=2,       # Seasonal orders
    D=1,                    # Seasonal differencing
    information_criterion='aic'
)

print("\nBest model:")
print(auto_model.summary())

# Extract best parameters
order = auto_model.order
seasonal_order = auto_model.seasonal_order
print(f"\nOrder (p,d,q): {order}")
print(f"Seasonal order (P,D,Q,m): {seasonal_order}")
```

**Example output:**
```
Order (p,d,q): (1, 1, 1)
Seasonal order (P,D,Q,m): (1, 1, 1, 7)
```

<Tip>
**SARIMAX parameter interpretation:**
- **p, d, q**: Non-seasonal AR, differencing, MA orders
- **P, D, Q, m**: Seasonal AR, differencing, MA orders, and period
- **m=7**: Weekly seasonality (7 days)
</Tip>

### Rolling Forecast with Retraining

Implement one-step-ahead rolling forecast:

```python
def rolling_forecast_sarimax(
    y_train, X_train,
    y_test, X_test,
    order, seasonal_order,
    retrain=True,
    retrain_period=30
):
    """
    Perform rolling one-step-ahead forecast with optional retraining
    """
    predictions = []
    actuals = []
    
    # Initial model training
    model = SARIMAX(
        y_train,
        exog=X_train,
        order=order,
        seasonal_order=seasonal_order,
        enforce_stationarity=False,
        enforce_invertibility=False
    )
    res = model.fit(disp=False, maxiter=200)
    
    # Rolling forecast
    for i in range(len(y_test)):
        # One-step-ahead prediction
        exog_future = X_test.iloc[i:i+1]
        pred = res.forecast(steps=1, exog=exog_future)[0]
        
        predictions.append(pred)
        actuals.append(y_test.iloc[i])
        
        # Retrain model periodically
        if retrain and (i + 1) % retrain_period == 0:
            # Append new data to training set
            y_extended = pd.concat([y_train, y_test.iloc[:i+1]])
            X_extended = pd.concat([X_train, X_test.iloc[:i+1]])
            
            # Refit model
            model = SARIMAX(
                y_extended,
                exog=X_extended,
                order=order,
                seasonal_order=seasonal_order,
                enforce_stationarity=False,
                enforce_invertibility=False
            )
            res = model.fit(disp=False, maxiter=200)
            print(f"Retrained at step {i+1}/{len(y_test)}")
    
    return np.array(predictions), np.array(actuals)

# Run rolling forecast
predictions, actuals = rolling_forecast_sarimax(
    y_train, X_train,
    y_test, X_test,
    order=order,
    seasonal_order=seasonal_order,
    retrain=True,
    retrain_period=30
)
```

### Model Evaluation

Calculate performance metrics:

```python
# Calculate metrics
rmse = np.sqrt(mean_squared_error(actuals, predictions))
mae = mean_absolute_error(actuals, predictions)
mape = np.mean(np.abs((actuals - predictions) / (actuals + 1e-10))) * 100

print("\n📊 Model Performance:")
print(f"RMSE: {rmse:.4f}")
print(f"MAE: {mae:.4f}")
print(f"MAPE: {mape:.2f}%")

# Create results dataframe
df_results = pd.DataFrame({
    'date': test_data['date'].values,
    'actual': actuals,
    'predicted': predictions,
    'error': actuals - predictions
})

```

Forecast both cases and deaths simultaneously:

## Best Practices

<Tip>
**Tips for better forecasting:**
1. **Cross-validation**: Use time series cross-validation for parameter tuning
2. **Exogenous variables**: Test different combinations of features
3. **Differencing**: Apply appropriate differencing to achieve stationarity
4. **Outlier handling**: Consider robust methods or outlier detection
5. **Ensemble methods**: Combine multiple models for improved accuracy
</Tip>

## Interpretation

Key findings from SARIMAX modeling:

- **Lagged cases** are strong predictors of deaths (typically 2-3 weeks delay)
- **Vaccination coverage** shows negative correlation with death rates
- **Stringency measures** have delayed effects on reducing deaths
- **Weekly seasonality** captures reporting patterns (fewer reports on weekends)
