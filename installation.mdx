This guide walks you through installing and setting up the COVID-19 Data Analytics project on your local machine.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Python 3.8 or higher** (Python 3.9-3.10 recommended)
- **Git** for cloning the repository
- **pip** package manager
- **4GB+ RAM** for running machine learning models
- **500MB+ disk space** for datasets and dependencies

<Tip>
  Check your Python version with `python --version` or `python3 --version`
</Tip>

## Installation Methods

<CodeGroup>

```bash macOS/Linux
# Clone the repository
git clone https://github.com/carlosperman/covid19-data-analytics-owid.git
cd covid19-data-analytics-owid

# Create virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install --upgrade pip
pip install -r requirements.txt
```

```bash Windows
# Clone the repository
git clone https://github.com/carlosperman/covid19-data-analytics-owid.git
cd covid19-data-analytics-owid

# Create virtual environment
python -m venv venv
venv\Scripts\activate

# Install dependencies
pip install --upgrade pip
pip install -r requirements.txt
```

</CodeGroup>

## Step-by-Step Installation

<Steps>
  <Step title="Clone the repository">
    Get the latest version from GitHub:
    
    ```bash
    git clone https://github.com/yourusername/covid19-data-analytics.git
    cd covid19-data-analytics
    ```
  </Step>
  
  <Step title="Create a virtual environment">
    Isolate project dependencies:
    
    <CodeGroup>
    ```bash macOS/Linux
    python3 -m venv venv
    source venv/bin/activate
    ```
    
    ```bash Windows
    python -m venv venv
    venv\Scripts\activate
    ```
    </CodeGroup>
    
    Your terminal prompt should now show `(venv)` indicating the virtual environment is active.
  </Step>
  
  <Step title="Install dependencies">
    The project requires 130+ Python packages. Installation may take 5-10 minutes:
    
    ```bash
    pip install --upgrade pip
    pip install -r requirements.txt
    ```
    
    <Note>
      Key dependencies include:
      - `streamlit==1.48.1` - Web application framework
      - `pandas==2.1.4` - Data manipulation
      - `scikit-learn==1.4.2` - Machine learning
      - `pycaret==3.3.2` - AutoML library
      - `plotly==5.24.1` - Interactive visualizations
      - `tensorflow==2.20.0` - Deep learning models
    </Note>
  </Step>
  
  <Step title="Extract compressed datasets">
    The project includes compressed datasets to save GitHub storage. Extract them before use:
    
    ```bash
    # Navigate to the app's datasets directory
    cd covid_app/datasets/inputs
    
    # Extract all compressed files
    unzip owid-covid-data.zip
    unzip countries-lat-lon.zip
    unzip HDR25_Composite_indices_complete_time_series.zip
    
    # Return to project root
    cd ../../..
    ```
    
    <Warning>
      The main dataset directory at `./datasets/` also contains a large `inputs.zip` (20MB). Extract this if you plan to run the analysis notebooks:
      
      ```bash
      cd datasets/inputs
      unzip inputs.zip
      ```
    </Warning>
  </Step>
  
  <Step title="Verify installation">
    Test that everything is set up correctly:
    
    ```bash
    # Check Python packages
    pip list | grep streamlit
    pip list | grep pandas
    
    # Verify datasets are extracted
    ls -lh covid_app/datasets/inputs/
    ```
    
    You should see `.csv` files instead of just `.zip` files.
  </Step>
  
  <Step title="Launch the application">
    Start the Streamlit app:
    
    ```bash
    cd covid_app
    streamlit run app.py
    ```
    
    The application will automatically open in your default browser at `http://localhost:8501`
    
    <Tip>
      If the browser doesn't open automatically, manually navigate to the URL shown in the terminal.
    </Tip>
  </Step>
</Steps>

## Jupyter Notebook Setup

To run the analysis notebooks:

```bash
# Ensure Jupyter is installed (included in requirements.txt)
pip install jupyter

# Launch Jupyter
jupyter notebook
```

Available notebooks:
- `cleaning.ipynb` - Data cleaning and preprocessing
- `eda.ipynb` - Exploratory data analysis (large file: 5MB+)
- `clustering.ipynb` - Clustering analysis (large file: 3.6MB+)
- `models.ipynb` - SARIMAX predictive modeling

<Warning>
  Before running notebooks, update file paths in the code to match your local directory structure. Look for variables like `data_path` or `input_folder` at the beginning of each notebook.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="ImportError: No module named 'streamlit'">
    This means the virtual environment is not activated or dependencies weren't installed.
    
    **Solution:**
    ```bash
    source venv/bin/activate  # Or venv\Scripts\activate on Windows
    pip install -r requirements.txt
    ```
  </Accordion>
  
  <Accordion title="FileNotFoundError: Dataset not found">
    The datasets are still compressed and need to be extracted.
    
    **Solution:**
    ```bash
    cd covid_app/datasets/inputs
    unzip '*.zip'
    ```
  </Accordion>
  
  <Accordion title="MemoryError when loading large datasets">
    Some operations require significant RAM, especially with 5+ million rows.
    
    **Solution:**
    - Close other applications to free memory
    - Use the deployed web app instead for basic exploration
    - Consider using a machine with more RAM for full analysis
  </Accordion>
  
  <Accordion title="pip install fails with compilation errors">
    Some packages like `numba` or `scikit-learn` require compilation tools.
    
    **Solution (macOS):**
    ```bash
    xcode-select --install
    ```
    
    **Solution (Ubuntu/Debian):**
    ```bash
    sudo apt-get install build-essential python3-dev
    ```
    
    **Solution (Windows):**
    Install [Microsoft C++ Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)
  </Accordion>
  
  <Accordion title="Streamlit app shows blank page or errors">
    Browser cache or port conflicts can cause display issues.
    
    **Solution:**
    ```bash
    # Clear Streamlit cache
    streamlit cache clear
    
    # Run on a different port
    streamlit run app.py --server.port 8502
    ```
  </Accordion>
  
  <Accordion title="Python version incompatibility">
    Some packages may not support Python 3.11+ or Python 3.7-.
    
    **Solution:**
    Use Python 3.9 or 3.10:
    ```bash
    # Install pyenv to manage Python versions
    pyenv install 3.10.0
    pyenv local 3.10.0
    
    # Recreate virtual environment
    python -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    ```
  </Accordion>
</AccordionGroup>

## Updating the Project

To get the latest changes:

```bash
# Pull latest code
git pull origin main

# Update dependencies
pip install --upgrade -r requirements.txt

# Clear Streamlit cache
streamlit cache clear
```

## Verifying Your Installation

Run this quick verification script:

```python
# test_install.py
import streamlit as st
import pandas as pd
import plotly.express as px
import pycaret
from pathlib import Path

print("✓ All core packages imported successfully")

# Check for datasets
datasets_path = Path("covid_app/datasets/inputs")
if datasets_path.exists():
    files = list(datasets_path.glob("*.csv"))
    if files:
        print(f"✓ Found {len(files)} CSV datasets")
    else:
        print("⚠ No CSV files found - extract the .zip files")
else:
    print("⚠ Datasets directory not found")

print("\nInstallation verification complete!")
```

Run with:
```bash
python test_install.py
```
